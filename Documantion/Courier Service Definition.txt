2. PROJECT ABSTRACT
-------------------
The GoParcel Logistics Management System is a full-stack web application designed to digitize and automate the manual processes of a courier service provider. The system serves as a centralized Admin Portal that facilitates the booking of shipments, automated cost calculation, real-time status updates, and invoice generation. Built using Python (Flask) for the backend and SQLite for data management, the project features a modern, high-performance frontend utilizing Glassmorphism design principles to provide an immersive user experience.

3. PROBLEM STATEMENT
--------------------
Traditional courier management often relies on manual entry, paper-based receipts, and opaque pricing models. This leads to:
- Calculation errors in shipping costs.
- Difficulty in tracking the real-time status of parcels.
- Inefficiency in generating billing and invoices.
- Lack of data visualization for business insights.

GoParcel addresses these issues by providing a unified digital interface to manage the entire lifecycle of a shipment from booking to delivery.

4. PROJECT OBJECTIVES
---------------------
- Automation: To automate the calculation of shipping costs based on weight, distance, and GST.
- Visualization: To provide a visual dashboard for monitoring delivery statistics (In Transit, Delivered, Total).
- Transparency: To offer a visual timeline for tracking shipment status (Booked -> In Transit -> Out for Delivery -> Delivered).
- Documentation: To generate printable, professional A4 invoices dynamically.

5. TECHNOLOGY STACK
-------------------
- Frontend: HTML5, CSS3 (Advanced Animations & Glassmorphism), JavaScript (ES6).
- Backend: Python 3.x (Flask Framework).
- Database: SQLite3.
- Data Transport: AJAX / Fetch API (JSON).

6. FUNCTIONAL MODULES
---------------------

A. Authentication Module
- Secure Admin Login: The system is protected by a login gateway requiring specific credentials to access the main dashboard.
- Session Management: Uses browser session storage to maintain login state.

B. Dashboard & Analytics
- Statistical Overview: Displays real-time counters for Total Shipments, Active Transit, and Delivered items.
- Admin Profile: Displays administrator details and system status.

C. Shipment Booking Engine
- Data Entry: Captures Sender and Receiver details (Name, Address, Phone).
- Smart Pricing Algorithm: Automatically calculates the total cost using the formula: Total = (Base Rate + (Weight * 20) + (Distance * 10)) + 18% GST.
- Custom ID Generation: Generates unique Shipment IDs with a custom prefix (e.g., MB_17193...) for internal tracking.

D. Shipment Management & Operations
- CRUD Operations: Allows the admin to View, Edit, and Delete shipment records.
- Status Management: Dropdown interface to update shipment stages.

E. Live Tracking System
- Visual Timeline: A graphical progress bar that lights up based on the current status of the parcel.
- Search Functionality: Real-time lookup of shipments using the custom Shipment ID.

F. Invoicing Module
- Dynamic Bill Generation: Creates a detailed invoice displaying sender/receiver info, cost breakdown, and tax details.
- Print-Ready Format: Uses CSS media queries to format the invoice specifically for A4 printing, hiding UI elements during the print process.

7. USER INTERFACE (UI) HIGHLIGHTS
---------------------------------
- Glassmorphism Theme: Utilizes transparency, blur filters (backdrop-filter), and sophisticated gradients to create a futuristic look.
- Micro-Interactions: Features animated toast notifications for success/error messages and floating background shapes for visual depth.
- Responsiveness: Fluid layouts using Flexbox and CSS Grid to adapt to different screen sizes.

8. CONCLUSION
-------------
The GoParcel project successfully integrates robust backend logic with a highly aesthetic frontend. It streamlines logistics operations by reducing manual errors in pricing and data entry while providing critical tools like real-time tracking and instant invoicing, making it a viable prototype for small-to-medium logistics companies.
